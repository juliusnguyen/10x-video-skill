# Implementation Plan: VieNeu TTS Integration

Replace ITERA102 API with the local \`vieneu\` Python SDK for high-quality Vietnamese TTS.

## Phase 1: Environment Setup
- [ ] Install \`vieneu\` and its dependencies in the virtual environment.
  - \`pip install vieneu --extra-index-url https://abetlen.github.io/llama-cpp-python/whl/metal/\` (since we are on Darwin/macOS).
- [ ] Verify installation by running a small test script.

## Phase 2: Core Implementation
- [ ] Create \`scripts/tts_vieneu.py\`:
  - A CLI wrapper for the \`vieneu\` SDK.
  - Arguments: \`text_file\`, \`output_file\`, \`voice_id\` (optional), \`emotion\` (optional).
  - Should support reading from the \`.txt\` files in \`voices/\`.
- [ ] Implement voice selection logic:
  - Map existing providers (\`minimax\`, \`elevenlabs\`) or define new ones for \`vieneu\` presets.

## Phase 3: Pipeline Integration
- [ ] Update \`scripts/tts.sh\`:
  - Either replace its content or make it call \`scripts/tts_vieneu.py\`.
  - Maintain backward compatibility with the \`<N> [provider]\` interface if possible.
- [ ] Update \`GEMINI.md\` and \`SKILL.md\` documentation to reflect the new TTS requirement.

## Phase 4: Validation
- [ ] Generate a test audio for a dummy video script.
- [ ] Verify file format (should be \`.mp3\` or \`.wav\` compatible with the renderer).
- [ ] Check duration using \`ffprobe\` as per the pipeline rules.

## Phase 5: Cleanup
- [ ] Remove/Deprecate \`ITERA102\` environment variables if no longer needed.
- [ ] Clean up any temporary models or cache files.
